# 后端工程师题库（共 20 道）

请描述一次完整的 HTTP 请求在你熟悉的 Web 框架（如 Spring Boot / Express / FastAPI）中，
从入口到响应返回的整个生命周期，包括路由匹配、过滤器/中间件、依赖注入、业务执行、异常处理。

如何设计一个支持高并发的秒杀系统？请从限流、削峰、库存扣减、防超卖、
缓存预热、降级方案等角度给出整体方案，并说明各组件的协作关系。

请讲解数据库索引的数据结构（B+ 树）原理，
以及为什么大多数 OLTP 数据库选择 B+ 树而不是哈希索引或 LSM 树。

什么是数据库的事务隔离级别？

请描述 MySQL 中常见的索引失效场景，并说明如何用 EXPLAIN 排查慢查询。

Redis 的常用数据结构有哪些？请分别说明它们的底层实现与典型业务场景，
比如 String、Hash、List、Set、ZSet、Bitmap、HyperLogLog。

谈谈你对 Redis 缓存穿透、缓存击穿、缓存雪崩的理解，
分别给出对应的工程化解决方案。

请描述 Kafka 的核心概念（Topic、Partition、Offset、Consumer Group），
并说明它如何同时保证高吞吐与消息顺序性。

如何保证消息中间件的消息不丢失、不重复、有序？请结合一个具体的中间件（如 Kafka 或 RocketMQ）说明。

请说明分布式系统中的 CAP 理论，并解释 BASE 思想是如何作为 CAP 的工程化补充的。

如何实现分布式锁？请对比 Redis 实现（含 Redlock）和基于 ZooKeeper / etcd 的实现，
讲清各自的优劣与适用场景。

讲讲你理解的微服务架构，
微服务拆分的原则、服务间通信方式（同步 vs 异步）、以及服务治理需要解决的核心问题。

请描述一次线上接口 P99 突然飙升你会如何排查，
覆盖：监控面板查看 → 日志定位 → 链路追踪 → 资源/中间件状态 → 代码层 review 的完整路径。

什么是幂等性？在订单支付、消息消费、HTTP 接口等场景中，分别如何保证幂等？

请讲解 JWT 的工作原理，与传统的 Session 鉴权相比有哪些优劣？
JWT 续签和登出失效问题应如何处理？

数据库分库分表的常见方案有哪些？水平拆分后会带来哪些新的复杂度（如分布式事务、跨库 JOIN、全局 ID 等）？

请讲讲你对 Java/Go/Python 任一语言的内存模型与垃圾回收（GC）机制的理解，
具体到一种 GC 算法（如 G1、ZGC、CMS、Go 的三色标记）。

什么是 RPC？相比 RESTful HTTP 接口，RPC（如 gRPC、Dubbo、Thrift）有哪些优势与代价？

谈一下你设计或参与过的最复杂的一次后端架构，
重点描述其中遇到的核心技术难点以及你给出的解法（最好包含取舍权衡）。

如果业务要求一个接口的吞吐从 200 QPS 提升到 5000 QPS，
